The function of the Consumption Definitions is to return the quantity value for a Service Point based on

  • Meter Data (in the order of Actual, Profiled, Forecast) as defined time period
  • Service Point Attribute value
  • Days of period

The user interface is available on the Web Portal for end user to manage the definitions, and the following is the instructions on how to use the management

Location: Home › Rates › Consumption definitions

 

Header Attributes

Code

  • Naming convention: Type of value_Flow Type_Additional Setting_Period Definitions_Timezone
  • Type of value & Flow Type
    • This is depending on the Value Column selected. For instance, EXP_KWH has the 'USAGE' as type of value and 'EXP' as flow type
    • The following is the Type of value reference table
  • Additional Setting
    • This is a free text allowed for user to add extra description to the definition code so that it is more meaningful to what it is mean to do such as ROLLING, DAILY_AVG, FIVE_NOMINATED_CPD and TOP4
  • Period Definitions
    • The time definitions applied to the period definition code
    • Describe the smallest grouping to the largest. For example
      • WORKDAY_NOV-DEC_1100-1500_APR-OCT_1200-1600
        • The definition above is to use Workday on both Nov to Dec at 11:00 to 15:00 and Apr to Oct at 12:00 to 16:00
        • The workday applies to both different period of month, hence only need to mention once
      • NOV-DEC_WORKDAY_1100-1500_APR-OCT_WEEKEND_1200-1600
        • Using the same month and time as per the first example, however for this example the workday applies only for Nov to Dec at 11:00 to 15:00, and weekend for Apr to Oct at 12:00 to 16:00
      • WEEKDAY_2100-0700_1000-1600_WEEKEND_ALLTIME
        • This example definition for the Weekday for the time between 21:00 to next day 07:00 and 10:00 to 16:00, and all time on Weekend
  • Timezone
    • ST for Standard Time (no daylight saving) - AEST for example
    • DST for Daylighting Saving Time - AEDT for example
    • Use the actual timezone code if the "Force use dst zone" is nominated. For example, ACST or ACDT for Broken Hill, NSW

 

Notes

  • Describe the Period Definition in sentence so that it is easier to understand rather than trying to decode from the code

 

Value Column

  • A set of options for user to choose which type of value to be used to return the quantity

 

Is Rolling

  • When selected this option, then Start Date of reading the data will be extended back to 12 months from the End Date
  • Start Date = Current End Date - 12 months

 

Final Aggregation

  • Instruction on the method to aggregate the collected reads

 

Formula

  • Specify instructions, normally a more complex calculation/aggregation that couldn't be catered by the standard aggregation

 

Rounding Type

  • Method to round the final aggregated value before turning to the requested process
